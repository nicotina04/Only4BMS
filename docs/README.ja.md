7鍵？ いいえ、結構です。

このプロジェクトは、すべてのBMS譜面を**4鍵 (DFJK)**に**強制的に再配置**してプレイできるようにする、PygameベースのBMSドライバーです。

## プロジェクト概要

Only4BMSは、7鍵や14鍵といった複雑なBMS譜面を見て、「どうやってこれを全部演奏すればいいんだ？」と考える4鍵主義者のために作られました。

このプロジェクトは、Pygameを使用してBMSファイルを解析し、どんな形式の譜面でも**4レーンにリマッピング**して、キー音と共に楽しめる環境を提供します。

## 主な機能

**強制4鍵マッピング**: 数学的アルゴリズムを使用して、5鍵、7鍵、10鍵、14鍵の譜面を自動的に4レーン（D、F、J、K）に割り当てます。

**密度チェッカー**: 7鍵譜面が4鍵に統合された際のノーツ密度をチェックし、視覚化します。

## 🛠️ 技術スタック

**言語**: Python 3.x

**ライブラリ**: Pygame (サウンドおよびレン더リング)

**形式**: Be-Music Script (.bms, .bme, .bml)

## プレイ方法 (曲の追加と実行)

1. **ゲームを入手**: ビルド済みの `Only4BMS.exe` をダウンロードするか、自身でビルドしてください。
2. **起動**: 任意のフォルダから `Only4BMS.exe` を実行するだけです。
3. **`bms` フォルダ**: 起動時に、ゲームは実行ファイルと同じディレクトリに `bms` フォルダを自動的に作成します。曲が見つからない場合は、基本的なデモ曲を生成します。
4. **自身の曲を追加**: 
   - BMS/BME/BMLファイルと関連メディア（オーディオ/BGAビデオ）をダウンロードします。
   - `bms/` ディレクトリ内の個別サブフォルダに解凍します。
   - 構造例：
     ```text
     [Only4BMS.exeがあるディレクトリ]
     ├── Only4BMS.exe
     └── bms/
         ├── Awesome Track/
         │   ├── song.bms
         │   ├── audio.wav
         │   └── video.mp4
         └── Another Track/
             └── ...
     ```

## 🎵 コースモード (ローグライクトレーニング)

同じ曲に飽きましたか？自動生成されるエンドレストレーニングモード、**コースモード**に挑戦しよう！
- 毎回異なる譜面がランダムに生成されます。
- 入門者(BPM 80~110)、中級(BPM 120~160)、上級(BPM 160~200)の難易度。
- ランクが上がるにつれ、ロングノートやソフラン（BPM変化）のギミックが登場。
- 各ステージは約30秒。生き残って次のコースを迎え討て！

## 🤖 AI 訓練とマルチプレイヤー

Only4BMSは、**強化学習 (PPO)** を基盤としたAIマルチプレイヤーモードを提供しています。

### 仕組み
- AIは、手続き的に生成されたリズムトラックと著作権フリーのデモ曲を使用して `stable-baselines3` で訓練されます。
- **法的遵守**: 倫理基準を保証するため、公式リリースにはCI/CDプロセス中に *非公式* データのみで訓練されたモデルが含まれています。
- **難易度**:
  - **NORMAL**: 25,000ステップ訓練。高い精度を誇りますが、時折人間のようなミスをします。
  - **HARD**: 40,000ステップ訓練。完璧に近いタイミングとコンボ維持を見せます。

### ローカル訓練
自身でモデルを訓練する場合：
1. 依存関係のインストール: `pip install stable-baselines3 shimmy gymnasium torch`
2. 訓練スクリプトの実行: `python -m only4bms.ai.train`
3. 生成された `model_normal.zip` および `model_hard.zip` は `src/only4bms/ai/` に保存されます。

### 自動化された CI/CD
GitHub Actions ワークフローは、リリースごとに同梱の `Mock Song Demo` を使用してAIモデルを一から再訓練します。これにより、ユーザーに配布されるバイナリが常に「クリーン」で最適化されていることを保証します。
