7키? 아니요, 괜찮습니다. 

이 프로젝트는 모든 BMS 차트를 **4키 (DFJK)**로 **강제 재배치**하여 플레이할 수 있게 해주는 Pygame 기반의 BMS 드라이버입니다.

## 프로젝트 개요

Only4BMS는 7키나 14키와 같은 복잡한 BMS 차트를 보고 "이걸 어떻게 다 연주하라는 거지?"라고 생각하는 4키 순수주의자들을 위해 만들어졌습니다.

이 프로젝트는 Pygame을 사용하여 BMS 파일을 파싱하고, 어떤 차트 형식이라도 **4개 레인으로 리매핑**하여 키 사운드와 함께 즐길 수 있는 환경을 제공합니다.

## 주요 기능

**강제 4키 매핑**: 수학적 알고리즘을 사용하여 5키, 7키, 10키, 14키 차트를 4개 레인(D, F, J, K)에 자동으로 할당합니다.

**밀도 체크**: 7키 차트가 4키로 통합될 때의 노트 밀도를 체크하고 시각화합니다.

## 🛠️ 기술 스택

**언어**: Python 3.x

**라이브러리**: Pygame (사운드 및 렌더링)

**형식**: Be-Music Script (.bms, .bme, .bml)

## 플레이 방법 (곡 추가 및 실행)

1. **게임 받기**: 빌드된 `Only4BMS.exe`를 다운로드하거나 직접 빌드하세요.
2. **실행**: 어느 폴더에서나 `Only4BMS.exe`를 실행하면 됩니다.
3. **`bms` 폴더**: 실행 시 게임은 실행 파일과 동일한 디렉토리에 `bms` 폴더를 자동으로 생성합니다. 곡이 발견되지 않으면 기본적인 데모 곡을 생성합니다.
4. **본인의 곡 추가**: 
   - BMS/BME/BML 파일과 관련 미디어(오디오/BGA 비디오)를 다운로드합니다.
   - `bms/` 디렉토리 내부의 개별 하위 폴더에 압축을 풉니다.
   - 구조 예시:
     ```text
     [Only4BMS.exe가 있는 디렉토리]
     ├── Only4BMS.exe
     └── bms/
         ├── Awesome Track/
         │   ├── song.bms
         │   ├── audio.wav
         │   └── video.mp4
         └── Another Track/
             └── ...
     ```

## 🎵 코스 모드 (로그라이크 트레이닝)

- 매 판마다 새롭게 랜덤으로 악보가 생성되어 내려옵니다.
- 입문자(BPM 80~110), 중급(BPM 120~160), 고급(BPM 160~200) 단계별 난이도 제공.
- 등급이 오르면서 롱노트와 기믹(변속) 패턴이 기습적으로 등장합니다.
- 각 스테이지는 약 30초! 살아남아서 다음 코스를 맞이하세요!

## 🤖 AI 훈련 및 멀티플레이어

Only4BMS는 **강화 학습 (PPO)**을 기반으로 한 AI 멀티플레이어 모드를 제공합니다.

### 작동 원리
- AI는 절차적으로 생성된 리듬 트랙과 저작권 없는 데모 곡을 사용하여 `stable-baselines3`로 훈련됩니다.
- **법적 준수**: 윤리적 기준을 보장하기 위해, 공식 릴리스는 CI/CD 프로세스 중에 *비상업적* 데이터로만 훈련된 모델을 포함합니다.
- **난이도**:
  - **NORMAL**: 25,000단계 훈련. 높은 정확도를 보이지만 가끔 사람 같은 실수를 합니다.
  - **HARD**: 40,000단계 훈련. 거의 완벽한 타이밍과 콤보 유지를 보여줍니다.

### 로컬 훈련
직접 모델을 훈련하려면:
1. 종속성 설치: `pip install stable-baselines3 shimmy gymnasium torch`
2. 훈련 스크립트 실행: `python -m only4bms.ai.train`
3. 생성된 `model_normal.zip` 및 `model_hard.zip`은 `src/only4bms/ai/`에 저장됩니다.

### 자동화된 CI/CD
GitHub Actions 워크플로는 모든 릴리스마다 포함된 `Mock Song Demo`를 사용하여 AI 모델을 처음부터 다시 훈련합니다. 이를 통해 사용자에게 배포되는 바이너리가 항상 "깨끗하고" 최적화되도록 보장합니다.
